Multimodal AI Intake and Knowledge Agent

Industry

Enterprise Knowledge Management

Client

Enterprise Integration

Tech Stack

n8n, OpenAI, MongoDB, Supabase, LangChain

The Challenge

Many teams receive customer questions and assets (voice notes, images, PDFs, spreadsheets) through messaging channels like WhatsApp, but lack an automated, consistent way to parse the content, extract structured information, index it for search, and answer follow-ups with context. Manual handling is slow, error-prone, and doesn't scale.

The Solution

This n8n workflow automates end-to-end multimodal intake and retrieval: accepts messages from WhatsApp (text, audio, image, document), routes media by type, uses OpenAI multimodal APIs to transcribe audio and analyze images, extracts document content, normalises prompts, generates embeddings, and stores them in a vector index. A LangChain-style Knowledge Base Agent with short-term memory retrieves context and answers user queries via WhatsApp.

Multimodal Intake Architecture

Solution Overview: Multimodal Intake

Trigger: WhatsApp message ingestion with automatic media type detection and routing.

WhatsApp Trigger & Route Types: Receives webhook, inspects message type (text/audio/image/document), and routes to appropriate processing branch.

Media Download Pipeline: Gets WhatsApp media URLs and downloads raw bytes for audio, images, and documents.

Type-Specific Processing: OpenAI audio translate for voicemail, image analyze for visual content, and native extractors (PDF/XLS/XLSX) for documents.

Unified Prompt Mapping: Standardizes all input types into a canonical JSON prompt sent to the Knowledge Base Agent.

Key Components & Data Flow

Trigger: Structured extraction and embedding generation for vector storage.

Document Type Routing & Extraction: Switch node routes documents by mime type; specialized extractors handle PDF, XLS, XLSX, JSON, CSV with output mapped to unified JSON schema.

Embeddings & Vector Storage: OpenAI embeddings are generated and stored in MongoDB Atlas vector collection (data_index) for semantic retrieval.

Knowledge Base Agent (LangChain): Receives unified prompt, consults vector store via productDocs tool, references short-term memory buffer, and uses gpt-4o-mini to compose context-aware answers.

Short-Term Memory (memoryBufferWindow): Conversation history keyed per WhatsApp user ID enables agent to reference recent messages within the session.

Data Flow (Step-by-Step)

Trigger: End-to-end message processing from WhatsApp ingestion to response delivery.

Steps 1–3: Ingestion & Media Download: User sends WhatsApp message → Trigger + Route Types → Download media bytes if needed.

Steps 4–6: Content Extraction: Audio → OpenAI translate → text; Image → analyze → description; Document → type-route → extractor → structured data.

Step 7: Unified Prompt: All branches converge into single JSON: text (standardised), metadata (mime type, caption, sender ID).

Steps 8–10: Retrieval & Response: Agent queries vector store (MongoDB), generates answer using gpt-4o-mini with memory context, sends WhatsApp reply.

Return on Investment

Eliminates manual document parsing and Q&A triage. Enables field teams, support staff, and knowledge workers to submit assets via WhatsApp and receive instant, contextual answers. Searchable knowledge base built automatically from user submissions. Reduces support response time by 80%+ and enables 24/7 instant answer capability.
